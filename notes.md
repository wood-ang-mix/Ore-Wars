# Ore-Wars 即时战斗逻辑设计

## 一、游戏循环（Game Loop）

即时战斗与回合制的本质区别：**时间在流动**。

需要一个"心跳"驱动整个游戏状态更新。推荐方案：

- ViewModel 里跑一个协程，每隔 50ms（20 tick/秒）执行一轮逻辑更新
- 逻辑和渲染分离：逻辑层按固定频率跑，UI 层通过 StateFlow 被动刷新，不跟帧率绑定

每一轮 tick 执行：

| 步骤 | 说明 |
|---|---|
| 处理输入队列 | 把玩家这 50ms 内的点击、按键转换成动作 |
| 执行动作 | 攻击、使用技能、放置物品 |
| 更新状态 | 扣血、转移归属、触发特效 |
| 更新冷却 | 技能/物品的 CD 减一 |
| AI 决策 | 敌人/NPC 决定这轮做什么 |
| 推 StateFlow | UI 自动刷新 |

## 二、数据模型

### BlockState（方块战斗状态）

在现有 `Block` 基础上增加战斗相关字段：

- `ownerId` — 归属哪个玩家（null = 中立）
- `hp` / `maxHp` — 生命值
- `status` — 正常 / 燃烧中 / 护盾 / 冰冻
- `defenseBonus` — 相邻友方方块给的防御加成

### PlayerState（玩家状态）

- `nation` — 矿石国家（决定技能池）
- `resources` — 矿石资源（购买物品/升级）
- `items` — 携带物品列表
- `cooldowns` — 各技能剩余冷却 tick 数
- `ownedBlockCount` — 已占领方块数

### Action（动作指令）

所有玩家操作封装成 Action 进队列，游戏循环按序处理。多人模式下两个玩家的动作合并到同一队列，天然解决并发：

```
AttackAction(x, y, playerId)      — 攻击方块
SkillAction(skillType, target)     — 使用技能
ItemAction(itemType, x, y)         — 放置/使用物品
```

## 三、战斗流程

玩家点击敌方方块的完整链路：

```
点击 → 构造 AttackAction → 进队列 → 下一 tick 取出

1. 计算基础伤害
   基础伤害 = 玩家攻击力（由矿石类型决定）

2. 应用被动加成
   例：红石国玩家旁边有青金石国 → 伤害 × 1.2（附魔加成）

3. 应用主动技能
   例：激活 TNT → 伤害 50，并对相邻方块溅射

4. 减去方块防御
   目标方块每有一个相邻友方方块 → 减伤 X%

5. 扣血 → 判断归属转移
   hp ≤ 0 → ownerId 切换，播放占领动画
```

## 四、技能系统

用**数据驱动**而非每个技能一个类：

```
技能 = {
    名称: "TNT",
    类型: AOE,
    伤害: 50,
    半径: 1,
    冷却_ticks: 100,
    所属国家: 红石国,
    特效: 爆炸动画
}
```

### 三种核心技能

| 矿石 | 技能 | 效果 |
|---|---|---|
| 红石 | TNT | 范围爆炸伤害，对目标及周围方块一起扣血 |
| 煤炭 | 火攻 | 给目标挂燃烧状态，每 tick 自动扣血持续 3 秒 |
| 黄金 | 金苹果 | 恢复自身血量 + 短暂无敌 |

技能只负责**修改目标方块的 hp 或 status**，动画交给 UI 层。

## 五、国家间增益（阵营联动）

全局被动 buff，每 tick 结束时根据领地邻接关系刷新：

```
每 tick 结束时检查：
  如果玩家 A（红石国）的方块中有紧邻玩家 B（青金石国）的方块
  → 玩家 A 攻击力 × 1.15
```

实现：用 `Map<(增益国, 被增益国), 增益效果>` 的配置表。

## 六、资源经济

- 每个已占领方块每 N tick 产出 1 矿石
- 矿石用于购买物品（放地上当陷阱或增益方块）
- 矿石可用于"升级"方块（提高 hp 上限）
- 占领新方块 → 一次性矿石奖励

策略维度：花钱强化现有领地 vs 攒资源一波推。

---

## 七、AI 对手

单人模式下每 30 tick 做一次决策：

```
1. 扫描地图 → 找出可攻击的敌方/中立方块
2. 优先级排序：
   - 邻接自己领地的方块 → 最高优先级
   - 稀有矿石方块 → 次高
   - 对方血量低的方块 → 再次
3. 若技能 CD 好了 → 对血量最低的目标使用
4. 其余 tick：持续攻击当前目标
```

## 八、整体架构总览

```
┌─────────────────────────────────────┐
│  UI 层 (Compose)                    │
│  MapGrid → 显示方块颜色/归属/特效    │
│  点击回调 → 构造 Action → 进队列     │
└──────────────┬──────────────────────┘
               │ collectAsState()
┌──────────────▼──────────────────────┐
│  GameViewModel                      │
│  ├─ GameLoop 协程 (20 tick/s)       │
│  ├─ StateFlow<GameState>            │
│  └─ ActionQueue                     │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│  游戏逻辑层                          │
│  ├─ CombatSystem   (伤害计算)       │
│  ├─ SkillSystem    (技能效果)       │
│  ├─ EconomySystem  (资源产出)       │
│  ├─ AISystem       (敌人决策)       │
│  └─ StatusSystem   (燃烧/护盾等)    │
└─────────────────────────────────────┘
```

## 九、实现顺序建议

1. 搭 **GameState / BlockState / PlayerState** 数据类 ——地基
2. 实现 **GameLoop 协程骨架** ——空 tick 跑起来
3. 点击事件改成 **AttackAction 进队列** ——点方块能扣血
4. 实现 **基础战斗** ——扣血 → HP 归零 → 归属转移
5. 加 **技能系统** ——先做 TNT 验证架构
6. 加 **国家增益** ——红石 + 青金石联动
7. 加 **AI** ——单人模式有对手
8. 加 **资源经济** ——买物品、升级方块
